# 09 · 본 프로젝트의 접근 차별성

> FlowBack의 접근은 한 문장으로 압축할 수 있다.
> **"스테이블코인을 결제수단이 아닌, 환불·보상·재구매의 가치 흐름을 잇는 정산 인프라로 사용한다."**

## 기존 접근과 어떻게 다른가

### 1. "결제 후"가 1급 시민

기존 결제·PG 시스템은 결제(payment) 자체를 1급 시민으로 다루고, 그 뒤에 일어나는 환불·보상·재구매를 부속 기능으로 다룬다.

FlowBack은 정반대다. **결제 이후 발생하는 모든 가치 이동**(부분 환불, 지연 보상, VIP 리워드, 재구매 시 잔액 사용)을 1급 시민으로 다루며, 결제는 그 흐름의 시작점일 뿐이다.

### 2. ledger-level Escrow를 정산 락업으로 사용

이 프로젝트가 EVM이 아닌 XRPL을 택한 결정적인 이유는 **별도 스마트컨트랙트 없이** 원장 자체가 제공하는 `EscrowCreate/Finish/Cancel` 기능을 활용하기 때문이다. 컨트랙트 감사 리스크가 없고, 가스비 변동도 없다.

각 주문은 단순한 DB row가 아니라, 실제로 ledger 위에서 잠긴 자금을 가리키는 **on-chain artifact**다. 분쟁 발생 시 PG 콘솔 로그가 아니라 ledger explorer에서 검증한다.

### 3. 카드 환불 D+2~7을 ledger close 3~5초로 압축

| 단계 | 기존 (카드/PG) | FlowBack (XRPL) |
|------|--------------|---------------|
| 환불 신청 → 카드사 접수 | 1일 | 0초 |
| 카드사 → 매입사 정산 | 1~3일 | N/A |
| 매입사 → 발급사 정산 | 1~3일 | N/A |
| 발급사 → 카드 한도 반영 | 1일 | 0초 |
| **합계** | **3~7영업일** | **3~5초** (ledger close) |

### 4. RLUSD 스테이블코인 단일 통화

USD 기반 스테이블코인 RLUSD를 보상·리워드 통화로 채택해 환율 변동에 따른 환불 손실을 제거한다. 한국 셀러는 원화 매출을, 해외 구매자는 USD 잔액을 본다.

### 5. 단일 월렛 = 단일 멘탈 모델

구매자에게는 한 화면이 곧 "내 돈의 위치"다. 환불 예정, 보상 누적, 재구매 가능 잔액이 분리된 시스템이 아닌 하나의 잔액과 그 변동 이력으로 보인다.

## 차별점이 만들어내는 외부 효과

- **셀러**: 운영 도구 4개(PG + 선정산 + 쿠폰 + CRM)를 1개로 통합 → 운영 시간 30~45분/일 절감
- **구매자**: "내 돈이 어디 있는지" 즉답 가능 → CS 응대 건수 2.7회 → 0.5회 미만으로 감소 (가정)
- **분쟁 처리**: ledger explorer 링크 한 번으로 책임 소재 검증 가능
- **글로벌 확장**: 통화·국가별 환불 규정 차이를 RLUSD 단일 통화로 흡수

## 한 줄 요약 (신청서 양식)

> 스테이블코인을 단순 결제수단으로 사용하는 것이 아니라, 거래 이후 발생하는 가치 흐름을 연결하는 정산 인프라로 활용한다. 환불·보상·리워드·재구매 혜택을 하나의 월렛 경험으로 통합함으로써, 사용자는 돈이 어디에 있고 언제 사용할 수 있는지 명확히 알 수 있고, 셀러는 고객 보상과 재구매 전환을 하나의 운영 구조 안에서 관리할 수 있다.
