# 08 · 기존 솔루션의 한계

| 솔루션 카테고리 | 대표 사례 | 강점 | 한계 (FlowBack 관점) |
|---------------|---------|------|---------------------|
| **카드사 / 글로벌 PG** | Stripe, Toss Payments, 이니시스, Shopify Payments | 결제 인프라 안정성 | 환불 D+3~7영업일, 부분 환불 단위 제한, 보상·재구매 연계 미지원 |
| **선정산 핀테크** | 정산일·고위드·NICE 핀테크 | 셀러 자금 회전 | 별도 수수료 2~3%, 신용 심사, 글로벌 거래 미지원 |
| **플랫폼 쿠폰·포인트** | 쇼피파이 / 카페24 / 자사몰 자체 | 마케팅 통합 | 플랫폼 내부 한정, 사용자가 실제 가치로 체감 어려움, 외부 이동 불가 |
| **CRM·마케팅 자동화** | Mailchimp, Klaviyo, Channeltalk | CS 워크플로 | 데이터 동기화 별도 필요, 정산과 분리 |
| **블록체인 기반 P2P 송금** | Ripple ODL, Stellar Anchors | 빠른 cross-border | B2B 송금에 특화, 커머스 사후 정산 워크플로 부재 |
| **스마트컨트랙트 Escrow** | Ethereum / EVM 기반 escrow DApps | 자동화 가능 | 가스비 변동성, 컨트랙트 감사 비용, 평균 종결 시간 12초+ |

## 각 한계의 결과

- **카드사 환불**: "환불은 했는데 카드에 안 보여요"라는 CS 메일이 셀러 응대 시간의 30% 이상을 차지
- **선정산 핀테크**: 수수료 부담이 SMB 셀러의 마진을 추가로 잠식
- **플랫폼 쿠폰**: 구매자는 "쿠폰은 있는데 어디서 쓰지?"가 됨 → 사용률 50% 미만
- **EVM Escrow**: 가스비 변동성으로 인해 작은 금액(< $10) 환불은 경제성이 없음

## 가장 핵심적인 한계

위 솔루션 중 어느 것도 **"환불 + 보상 + 재구매 혜택 + 사용자 잔액"** 4개를 하나의 흐름으로 묶지 못한다. 그래서 셀러는 운영 도구를 늘려야 하고, 구매자는 자신의 돈이 어디에 있는지 모른 채로 다음 구매에 진입한다. FlowBack은 이 단절을 메우는 인프라다.
